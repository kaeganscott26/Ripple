# CHARACTER SCHEMA

_Status: Agent Memory Template | Human-Readable First | Engine-Ready Later_

Each Ripple character should be written as a memory substrate, not a normal character bio.

The agent should be playable because of what they remember, misunderstand, fear, and pass down.

---

## Required Fields

### Name

Character or symbolic agent name.

### Source Role

Where the character comes from in the copied source material or simulation design.

### Ripple Role

What the character does inside the game board.

### Identity Core

The part of the character that stays stable across Life A, Life B, and Life C.

### Life A

One possible remembered life.

### Life B

Second possible remembered life.

### Life C

Third possible remembered life.

### Primary Fear

What this agent interprets as danger.

### Primary Desire

What this agent moves toward.

### Primary Distortion

What this agent tends to misread.

### Trigger Objects

Artifacts, room types, phrases, or situations that strongly affect the agent.

### Ripple Behavior

How this agent spreads, resists, distorts, archives, or institutionalizes ripples.

### Inheritance Pattern

What this agent would pass down if they helped build a simulation inside the simulation.

---

## Runtime Compression

The full life files may be too long for runtime prompts.

Later, Codex should generate compact data from these documents:

- core identity summary
- active life seed
- top memories
- current beliefs
- current fears
- active relationships
- selected triggers
- current ripple interpretation

---

## Writing Rule

Do not write the life variants as fixed destiny.

A Life seed gives an agent a starting lens.

It does not force every choice.

The room, artifact, other agents, and current ripple still matter.

---

## Key Sentence

The player does not discover who the character is.

The player discovers which version of the character this reality remembered.
