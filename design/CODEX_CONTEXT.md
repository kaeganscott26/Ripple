# CODEX CONTEXT

_Status: Required Context | Read Before Code | Build Guardrails_

Codex should read this file before modifying the Ripple project.

Ripple is a standalone game repository derived from copied source material.

The copied INTERVENTION material is currently present as fuel for extraction, not as the final game structure.

---

## Highest-Level Instruction

Do not build a generic AI NPC demo.

Build toward a 2D living board game where memory, observation, artifacts, and agent interpretation generate emergent social reality.

---

## Do Not Do These Things

- Do not turn chapters directly into fixed levels.
- Do not hard-script the INTERVENTION story as gameplay.
- Do not delete source material unless asked.
- Do not claim the game proves reality is fake.
- Do not claim agents are literally conscious.
- Do not make paid AI APIs required.
- Do not begin with a huge 3D world.
- Do not make Minecraft the first target.

---

## Do These Things

- Treat source chapters as symbolic room DNA.
- Treat artifacts as active board objects.
- Treat characters as agents with memory variants.
- Treat layers as system rules or board pressures.
- Treat simulation logs as generated artifacts.
- Build deterministic rules before optional AI.
- Keep the game local-first.
- Prefer a 2D board, cards, panels, and logs.

---

## Core Build Order

1. Preserve source material.
2. Build design context.
3. Extract rooms and characters.
4. Define memory seeds.
5. Create data schemas.
6. Build a tiny rules-based board prototype.
7. Add event logs and reality layers.
8. Add optional local AI.
9. Add nested simulation mechanics.

## Runtime Principle

The game should not ask the model to invent everything live.

The repo should provide:

- source structure
- character memory
- artifact meaning
- room pressure
- rule boundaries

The engine should use those structures to generate play.

---

## Codex Role

Codex is not the author of Ripple.

Codex is the contractor that turns the source archive into a game skeleton.

The source material provides symbolic DNA.

The design documents provide direction.

The first prototype should prove the loop, not the whole vision.
