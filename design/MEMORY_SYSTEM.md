# MEMORY SYSTEM

_Status: Agent Substrate | Replayability System | Opening Mode Core_

Ripple begins with memory.

The map does not matter until someone remembers how to enter it.

---

## Core Rule

Characters do not have one fixed playable past.

Each character has:

- a stable identity core
- Life A
- Life B
- Life C
- optional fragments
- memories created during the current run

The selected life changes what the character believes, fears, notices, and passes down.

---

## Why This Exists

A fixed life history can become a cage.

If Mara always remembers the same wound, Mara may eventually make similar choices across every playthrough.

Ripple avoids this by giving each agent multiple possible remembered lives.

The same character remains recognizable, but the active memory seed changes how they interpret the board.

The player does not always know which version is active.

---

## Opening Modes

### Mystery

The board secretly chooses each agent's life.

The player discovers memory through behavior.

### Vague

The player chooses A, B, or C without seeing the full meaning.

The player chooses the beginning without knowing the consequence.

### Experimental

The player sees memory variants before choosing.

This allows deliberate testing.

---

## Memory Selection Language

Do not call this a standard character creator.

Use language like:

- The Remembering
- Memory Seed
- Remembered Life
- The Board Remembered Mara as B

Avoid language like:

- Backstory picker
- Character class
- Stat roll

---

## Memory Types

### Identity Core

The stable through-line of a character.

Example:

Mara is alert to pressure, sensitive to thresholds, and suspicious of commands disguised as comfort.

### Life Variant

The selected memory build for the run.

Example:

Mara A may remember scarcity.

Mara B may remember doors.

Mara C may remember false testimony.

### Memory Fragments

Small additional memories that may be added or revealed during play.

Fragments create variation without replacing the core life.

### Runtime Memory

Events that happen during the current simulation.

These must matter.

Agents should remember what the player did, what other agents said, what artifacts changed, and what laws formed.

---

## Memory Reveal

The player should not always see full memory at the start.

Memories can reveal through:

- agent reactions
- artifact inspection
- repeated behavior
- dialogue
- turn events
- reality layer shifts
- nested simulation inheritance

Memory reveal should feel like observation, not exposition dumping.

---

## Fractured Memory Option

Later versions may include rare fractured memory states.

A fractured agent may carry pieces of multiple life variants.

This should increase:

- contradiction
- observer suspicion
- pattern detection
- emotional instability
- nested simulation risk

This is not required for the first prototype.

---

## Nested Simulation Inheritance

When agents build an inner simulation, they do not pass down source truth.

They pass down their active remembered lives, beliefs, fears, laws, artifacts, and distortions.

A deeper civilization may inherit an artifact as a myth, a myth as a law, and a law as physics.

No layer can transmit what it cannot observe.
