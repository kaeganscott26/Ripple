# ROOM SYSTEM

_Status: Symbolic Room Design | Source Extraction Rule_

Ripple does not convert stories into fixed levels.

Ripple extracts symbolic room functions from source material and lets those functions generate new outcomes.

---

## Core Rule

The stories are not levels.

The stories are fossils.

A chapter provides symbolic DNA.

Ripple grows a room from that DNA.

---

## Room Definition

A Ripple room should include:

- source reference
- symbolic name
- room function
- pressure type
- primary artifact
- active agents
- possible ripple inputs
- reality-layer risk
- possible emergent institutions
- possible nested inheritance

---

## Example: Boulder Room

Source function:

Weight / consequence made visible.

Room artifact:

Boulder.

Pressure:

Consequence, obstruction, naming, weight, delayed recognition.

Possible outcomes:

- boulder becomes a joke
- boulder becomes a warning
- boulder blocks a path
- boulder is named and becomes law
- boulder becomes origin myth in a nested simulation

The room should not replay a fixed chapter.

It should allow new meanings to form.

---

## Room Types

### Memory Room

Changes what agents remember or reveal.

### Pressure Room

Raises emotional or social pressure.

### Decision Room

Forces a choice or fork.

### Echo Room

Repeats old patterns in distorted form.

### Artifact Room

Centers play around one symbolic object.

### Theory Room

Unlocks understanding through mechanics.

### Closed Door Room

Requires naming, witness, or changed conditions before passage.

### Simulation Room

Allows agents to model the world or create nested simulations.

---

## Room UI

A room panel should eventually show:

- room title
- room function
- active artifact
- active agents
- current pressure
- current ripple
- current law or institution
- base reality
- perceived reality
- social reality
- institutional reality

---

## Room Outcome Rule

A room should never have only one correct outcome.

The room has a function.

The playthrough creates the outcome.
